EEG-Pype: An accessible MNE-Python pipeline with graphical user interface for preprocessing and analysis of resting-state electroencephalography data

PLOS Computational Biology | https://doi.org/10.1371/journal.pcbi.1014043 March 2, 2026

Citation: Lodema DY, van Dellen HJ, de Haan W, van Hest M, Hillebrand A, van Dellen E (2026) EEG-Pype: An accessible MNE-Python pipeline with graphical user interface for preprocessing and analysis of resting-state electroencephalography data. PLoS Comput Biol 22(3): e1014043.

Authors: D. Yorben Lodema, Herman J. van Dellen, Willem de Haan, Margot van Hest, Arjan Hillebrand, Edwin van Dellen (University Medical Center Utrecht; Amsterdam UMC; Vrije Universiteit; UZ Brussel).

ABSTRACT
Processing of electroencephalography (EEG) data requires multiple steps to remove noise and artifacts and select good-quality data. While powerful open-source toolboxes like MNE-Python exist, their command-line nature can pose a barrier for researchers without programming experience. Here, we present EEG-Pype, an open-source (Apache-2.0 licensed) graphical user interface application using MNE-Python functions. EEG-Pype provides an intuitive workflow tailored for preprocessing of resting-state EEG data, including frequency band filtering, independent component analysis and atlas-based beamforming for source-level analysis. The application supports several common raw EEG input file formats and guides users through a comprehensive pipeline focused on manual bad channel and epoch selection. Manual steps are streamlined using MNE-Python's interactive plots. Configuration saving and loading allows for batch (re)runs, while a separate log is saved, improving reproducibility. Output can be saved after filtering in canonical frequency bands. EEG-Pype includes a module for calculating quantitative EEG measures on preprocessed data, including spectral, functional connectivity and network analysis metrics. Download: https://github.com/yorbenlodema/EEG-Pype

KEY POINTS FROM PAPER:
- Motivation: MNE-Python is powerful/free/open but command-line; EEGLAB, Brainstorm, FieldTrip are MATLAB (proprietary); BrainWave/DIGEEG (C.J. Stam) were Java, single-developer. Need modern open-source successor.
- Workflow philosophy: resting-state EEG, manual data selection central; ICA optional; semi-rigid pipeline trade-off between ease-of-use and customizability.
- Pipeline stages: (1) Data Input - batch, formats BDF/EDF/FIF/BrainVision/CNT/ASCII/EEGLAB/EGI; (2) Preprocessing - visual PSD inspection, drop channels, manual bad channel rejection for interpolation, ICA (default 25 components, 1.0-47.0 Hz bandpass copy, MNE-ICALabel for auto labeling brain/eye/muscle/heart/line noise), spherical spline interpolation, broadband filtering, optional average referencing; (3) Epoching - user-defined length, manual visual rejection; (4) Source Analysis - optional LCMV beamformer, fsaverage template MRI, Desikan-Killiany atlas (68 regions), Tikhonov regularization lambda=0.05, unit-noise-gain normalization; (5) Data Export - clean epochs/continuous, filter into delta/theta/alpha/beta bands, log files + .pkl batch files; (6) Quantitative Analysis - PSD (Multitaper/Welch/FFT), absolute/relative power, peak frequency, spectral variability; entropy (permutation, sample, approximate); functional connectivity (PLI, corrected AEC with orthogonalization, JPE); network topology via minimum spanning tree (Kruskal algorithm).
- Results: demonstrated on 64-channel BioSemi .BDF sample (~10 min). Broadband FIR filter 0.5-47 Hz removed 50 Hz line noise. Dropped 9 non-EEG channels -> 64 analyzable. ICA identified 2 frontal eye-blink components (ICA000, ICA004). 8-second epochs: 84 initial, 29 rejected, 55 kept. Export: 330 files (55 epochs x 6 bands: broadband, delta, theta, alpha, beta1, beta2). Total preprocessing ~2 minutes.
- Example whole-brain metrics: peak frequency 9.18 Hz; theta relative power 0.224, alpha 0.170; various entropy and connectivity values reported.
- Discussion: compared to Meggie and MNELAB (other MNE GUIs) - EEG-Pype provides complete pipeline without further software, central manual epoch selection, resting-state focus. Inspired by BrainWave. Limitations: semi-rigid, less customization; template-based beamforming and diagonal noise covariance are simplifications reducing precision vs individual MRI best practices.
- License: Apache-2.0. Available on GitHub with video tutorial. Development expected to continue with MNE-Python.
